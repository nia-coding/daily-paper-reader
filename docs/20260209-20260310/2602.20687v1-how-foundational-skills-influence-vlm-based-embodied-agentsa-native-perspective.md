## 摘要
视觉语言模型（VLM）的最新进展为实现人类水平的具身智能带来了希望。然而，现有的 VLM 驱动具身智能体基准测试通常依赖于高层指令或离散化的动作空间，这些非原生设置与现实世界的控制存在显著差异。此外，目前的基准测试主要关注高层任务，缺乏对底层和高层任务的联合评估与分析。为了解决这些局限性，我们提出了 NativeEmbodied，这是一个针对 VLM 驱动具身智能体的具有挑战性的基准测试，它使用统一的原生底层动作空间。NativeEmbodied 构建于多样的模拟场景之上，包含复杂场景中的三个代表性高层任务，用以评估整体性能。为了进行更详细的分析，我们进一步解耦了复杂任务所需的技能，并构建了四类底层任务，每类任务针对一种基础具身技能。这种跨任务和技能粒度的联合评估实现了对具身智能体的细粒度评估。对最先进 VLM 的实验揭示了其在几种基础具身技能上的明显缺陷，进一步分析表明，这些瓶颈显著限制了其在高层任务上的表现。NativeEmbodied 突出了当前 VLM 驱动具身智能体面临的关键挑战，并为指导未来研究提供了见解。

## Abstract
Recent advances in vision-language models (VLMs) have shown promise for human-level embodied intelligence. However, existing benchmarks for VLM-driven embodied agents often rely on high-level commands or discretized action spaces, which are non-native settings that differ markedly from real-world control. In addition, current benchmarks focus primarily on high-level tasks and lack joint evaluation and analysis at both low and high levels. To address these limitations, we present NativeEmbodied, a challenging benchmark for VLM-driven embodied agents that uses a unified, native low-level action space. Built on diverse simulated scenes, NativeEmbodied includes three representative high-level tasks in complex scenarios to evaluate overall performance. For more detailed analysis, we further decouple the skills required by complex tasks and construct four types of low-level tasks, each targeting a fundamental embodied skill. This joint evaluation across task and skill granularities enables fine-grained assessment of embodied agents. Experiments with state-of-the-art VLMs reveal clear deficiencies in several fundamental embodied skills, and further analysis shows that these bottlenecks significantly limit performance on high-level tasks. NativeEmbodied highlights key challenges for current VLM-driven embodied agents and provides insights to guide future research.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现有的视觉语言模型（VLM）具身智能体评测基准存在两个主要缺陷：
    1.  **非原生动作空间（Non-Native Action Space）**：大多依赖高层抽象指令（如“传送到桌子旁”），忽略了现实世界中复杂的底层控制（如精确的移动距离和旋转角度）。
    2.  **任务耦合（Coupled Task Design）**：现有评测多关注高层复杂任务的成功率，难以诊断究竟是哪种底层基础技能（如感知、导航或空间对齐）制约了智能体的表现。
*   **研究动机**：为了弥合模拟环境与现实控制之间的差距，并深入探究 VLM 在具身任务中的性能瓶颈。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：提出 **NativeEmbodied** 基准，采用统一的、原生的底层动作空间，并通过“自底向上”的解耦设计，将高层复杂任务拆解为可独立评估的底层基础技能。
*   **关键技术细节**：
    *   **原生动作空间（Native Action Space）**：定义了包括 `MoveAhead x`、`RotateRight x`、`LookDown x` 等在内的原始动作原语，允许智能体自定义参数（距离、角度），而非调用预设函数。
    *   **解耦任务层次结构**：
        *   **高层任务（High-level）**：探索（Exploration）、搜索（Search）、交互（Interaction）。
        *   **底层技能（Low-level）**：感知（Perception）、空间对齐（Spatial Alignment）、导航（Navigation）、规划（Planning）。
    *   **数据生成管线**：利用 AI2THOR 模拟器的元数据自动生成候选样本，并通过“人机协作过滤”机制（MLLM 预跑 5 轮 + 人工审核）确保样本的挑战性和可行性。

### 3. 实验设计
*   **场景与数据集**：基于 **AI2THOR** 模拟器，包含 1,085 个测试样本。
*   **对比方法（Baselines）**：评估了 15 种主流开源和闭源模型，涵盖四个家族：
    *   **GPT 系列**：GPT-4o, GPT-o3, GPT-4v 等。
    *   **Claude 系列**：Claude-3.5/3.7 Sonnet, Claude-4 Opus 等。
    *   **Gemini 系列**：Gemini-2.5 Pro/Flash 等。
    *   **Qwen 系列**：Qwen2.5-VL (72B/32B/7B/3B)。
*   **评测指标**：除了成功率（SR），还引入了平均步数（AS）、加权平均步数（WAS）、平均最近距离（ACD）和平均最近像素距离（ACPD）等细粒度指标。

### 4. 资源与算力
*   **算力说明**：论文主要侧重于对现有预训练 VLM 的**推理评测**，而非模型训练。
*   **推理设置**：所有模型推理时温度（Temperature）设为 0 以保证可重复性。文中未明确提及具体的 GPU 型号或总推理时长，但考虑到评估了 15 个模型且包含多轮 Rollout，其推理开销主要集中在 API 调用和大规模模拟器运行上。

### 5. 实验数量与充分性
*   **实验规模**：
    *   **主实验**：在 7 类任务（3 高 + 4 低）上对 15 个模型进行了全面测试。
    *   **消融实验**：针对基础技能（提供 Ground-truth 信息）和“思考模式（Think Mode）”进行了专项消融。
*   **充分性与客观性**：实验覆盖了当前最顶尖的闭源模型和不同规模的开源模型，对比维度丰富。通过解耦实验，客观地揭示了模型在不同技能上的长短板，避免了单一成功率指标的片面性。

### 6. 主要结论与发现
*   **原生环境挑战巨大**：即使是最强的模型（如 GPT-o3），在原生设置下的搜索任务成功率也仅为 34.9%，说明当前 VLM 距离真正的具身控制仍有很大差距。
*   **底层技能瓶颈**：**空间对齐（Alignment）**和**导航（Navigation）**是当前最显著的短板。大多数模型在空间对齐上的成功率甚至不足 50%。
*   **思考模式的“双刃剑”效应**：开启“思考模式”能显著提升感知和规划等认知能力，但可能会干扰底层的动作执行，导致导航和对齐性能下降。
*   **常见错误模式**：总结了探索不足、冗余视角调整和频繁碰撞三大典型错误。

### 7. 优点（亮点）
*   **视角独特**：首次从“原生动作空间”角度审视 VLM 具身智能体，更贴近真实物理世界的控制逻辑。
*   **诊断性强**：通过解耦高低层任务，能够精准定位模型是“脑子不行（规划）”还是“手脚不灵（控制）”。
*   **评测全面**：涵盖了从感知到执行的全链路技能，且对比模型数量多，具有很强的行业参考价值。

### 8. 不足与局限
*   **环境单一性**：实验主要局限于 AI2THOR 室内场景，尚未扩展到室外或更复杂的工业环境。
*   **Sim-to-Real 差距**：虽然采用了原生动作，但模拟器中的物理碰撞和摩擦力模型与现实世界仍有差异。
*   **样本规模**：1,085 个样本虽然质量高，但对于覆盖极其复杂的具身交互场景来说，规模仍有提升空间。
