## 摘要
具身智能体的真实世界数据采集仍然成本高昂且不安全，因此需要可扩展、逼真且可直接用于模拟器的 3D 环境。然而，现有的场景生成系统通常依赖于基于规则或特定任务的流水线，从而产生伪影和物理上无效的场景。我们提出了 SAGE，这是一个智能体框架，在给定用户指定的具身任务（例如“拿起碗并将其放在桌子上”）时，能够理解意图并自动大规模生成可用于模拟的环境。该智能体将用于布局和物体组合的多个生成器与评估语义合理性、视觉逼真度和物理稳定性的评论器相结合。通过迭代推理和自适应工具选择，它会对场景进行自我优化，直到满足用户意图和物理有效性。生成的环境逼真、多样，且可直接部署在现代模拟器中进行策略训练。完全基于此类数据训练的策略表现出明显的扩展趋势，并能泛化到未见过的物体和布局，证明了模拟驱动扩展在具身智能领域的潜力。代码、演示和 SAGE-10k 数据集可在项目主页找到：https://research.nvidia.com/labs/dir/sage/。

## Abstract
Real-world data collection for embodied agents remains costly and unsafe, calling for scalable, realistic, and simulator-ready 3D environments. However, existing scene-generation systems often rely on rule-based or task-specific pipelines, yielding artifacts and physically invalid scenes. We present SAGE, an agentic framework that, given a user-specified embodied task (e.g., "pick up a bowl and place it on the table"), understands the intent and automatically generates simulation-ready environments at scale. The agent couples multiple generators for layout and object composition with critics that evaluate semantic plausibility, visual realism, and physical stability. Through iterative reasoning and adaptive tool selection, it self-refines the scenes until meeting user intent and physical validity. The resulting environments are realistic, diverse, and directly deployable in modern simulators for policy training. Policies trained purely on this data exhibit clear scaling trends and generalize to unseen objects and layouts, demonstrating the promise of simulation-driven scaling for embodied AI. Code, demos, and the SAGE-10k dataset can be found on the project page here: https://research.nvidia.com/labs/dir/sage/.

### 2. 方法论
SAGE 采用了一种**基于智能体（Agentic）的框架**，核心思想是通过“生成器-评估器”的闭环迭代来优化场景。
*   **核心架构**：基于模型上下文协议（MCP），将 Agent 作为客户端，协调多个外部工具（服务器）。
*   **关键技术组件**：
    *   **生成器（Generators）**：包括场景初始化（生成户型图）、资产放置（利用 TRELLIS 进行文本转 3D 物体生成）、资产移动和移除工具。
    *   **视觉评估器（Visual Critic）**：利用多视图渲染和 VLM（视觉语言模型）评估场景的语义合理性、布局完整性和视觉真实感。
    *   **物理评估器（Physics Critic）**：**核心创新点**。将 Isaac Sim 模拟器引入生成循环，实时检测物体的碰撞和重力稳定性。若物体在模拟中倒塌或移位，则反馈给 Agent 进行修正。
*   **数据增强与动作生成**：
    *   **多级增强**：在物体配置、类别（改变形状/纹理）和场景布局层面进行增强，实现从单个种子场景扩展出海量变体。
    *   **自动标注**：利用 M2T2 进行抓取预测，Curobo 进行逆运动学（IK）求解，RRT 进行路径规划，自动生成专家演示数据。

### 3. 实验设计
*   **场景生成实验**：
    *   **常见场景**：卧室、厨房、客厅。
    *   **开放词汇场景**：健身房、办公室、赛博朋克游戏室、星空主题卧室等。
    *   **对比基线**：Holodeck（基于 LLM 的静态流水线）和 SceneWeaver（基于 Agent 但缺乏物理验证）。
*   **具身智能任务**：
    *   **取放任务（Pick-and-Place）**：使用 Franka 机械臂。
    *   **移动操作（Mobile Manipulation）**：机械臂搭载在移动底座上，执行跨房间的导航与物体搬运。
*   **数据集**：发布了 **SAGE-10k**，包含 50 种房间类型、50 种风格、共 1 万个场景和 56.5 万个唯一生成的 3D 物体。

### 4. 资源与算力
*   **硬件环境**：实验主要在 **NVIDIA L40S GPU** 集群上运行。
*   **生成效率**：
    *   **场景生成**：生成一个包含 20 个物体的场景约需 10 分钟（物体生成通过 8 张 GPU 并行加速至每个物体 2-3 秒）。
    *   **动作生成**：利用 Isaac Lab 的并行能力，取放任务的专家演示生成速度可达每条 2-3 秒。
*   **模型使用**：Agent 采用 `gpt-oss-120b`，视觉推理采用 `Qwen3-VL-30B`。

### 5. 实验数量与充分性
*   **实验规模**：
    *   生成了 10,000 个场景进行评估。
    *   收集了超过 2.8 万条取放任务演示和 5 万条移动操作演示。
*   **消融实验**：针对“视觉评估器”和“物理评估器”分别做了对比，证明了物理模拟在环（In-the-loop）对提升场景稳定性的关键作用（稳定性从 80.3% 提升至 99.6%）。
*   **客观性**：引入了跨环境评估（Cross-Evaluation），在基线方法生成的场景上测试 SAGE 训练的策略，证明了其更强的泛化能力。实验设计覆盖了从感知到执行的全流程，论证较为充分。

### 6. 主要结论与发现
*   **物理验证至关重要**：没有物理评估器的 Agent 容易生成视觉美观但物理失效（如物体嵌入桌面）的场景，导致机器人无法学习。
*   **规模化效应（Scaling Law）**：随着生成场景数量和演示数据量的增加，机器人策略的成功率呈现明显的上升趋势，且逐渐逼近拥有特权信息的专家（Privileged Agent）水平。
*   **泛化性**：在 SAGE 生成的合成数据上训练的策略，能够直接泛化到未见过的物体几何形状和复杂的房间布局中。

### 7. 优点
*   **闭环自优化**：不同于传统的单向生成流水线，SAGE 能通过反馈自我修正错误。
*   **模拟就绪性**：生成的场景自带物理属性（质量、摩擦力、PBR 材质），可直接用于强化学习或模仿学习。
*   **开放词汇能力**：结合文本转 3D 技术，摆脱了传统方法依赖有限资产库的限制，极大提升了场景多样性。

### 8. 不足与局限
*   **物理类型局限**：目前主要集中在室内场景和刚体物理，尚未深入涉及室外环境、柔性物体（如布料）或复杂的流体交互。
*   **生成耗时**：尽管有并行加速，但生成一个高质量场景仍需分钟级时间，对于超大规模实时生成仍有挑战。
*   **任务复杂度**：目前的动作生成主要基于取、放、导航的组合，对于更复杂的长程逻辑任务（如烹饪、清洁）的自动生成仍待探索。
