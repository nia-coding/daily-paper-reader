## 摘要
快速演变的网络攻击要求事件响应系统能够自主学习并适应不断变化的威胁。先前的工作广泛探索了强化学习方法，该方法涉及通过对事件进行大量模拟来学习响应策略。虽然这种方法可能有效，但它需要对手工建模模拟器，并抑制了原始系统日志和警报中有用的语义信息。为了解决这些局限性，我们建议利用大语言模型（LLM）预训练的安全知识和上下文学习（in-context learning），为事件响应规划创建一种端到端的智能体解决方案。具体而言，我们的智能体将感知、推理、规划和行动四种功能集成到一个轻量级 LLM（14b 模型）中。通过微调和思维链（chain-of-thought）推理，我们的 LLM 智能体能够处理系统日志并推断潜在的网络状态（感知），更新其对攻击模型的推测（推理），模拟不同响应策略下的后果（规划），并生成有效的响应（行动）。通过将 LLM 模拟的结果与实际观察结果进行比较，LLM 智能体不断完善其攻击推测和相应的响应，从而展示了上下文自适应能力。我们的智能体方法无需建模，且可以在通用硬件上运行。在对文献中报告的事件日志进行评估时，我们的智能体实现的恢复速度比前沿 LLM 快 23%。

## Abstract
Rapidly evolving cyberattacks demand incident response systems that can autonomously learn and adapt to changing threats. Prior work has extensively explored the reinforcement learning approach, which involves learning response strategies through extensive simulation of the incident. While this approach can be effective, it requires handcrafted modeling of the simulator and suppresses useful semantics from raw system logs and alerts. To address these limitations, we propose to leverage large language models' (LLM) pre-trained security knowledge and in-context learning to create an end-to-end agentic solution for incident response planning. Specifically, our agent integrates four functionalities, perception, reasoning, planning, and action, into one lightweight LLM (14b model). Through fine-tuning and chain-of-thought reasoning, our LLM agent is capable of processing system logs and inferring the underlying network state (perception), updating its conjecture of attack models (reasoning), simulating consequences under different response strategies (planning), and generating an effective response (action). By comparing LLM-simulated outcomes with actual observations, the LLM agent repeatedly refines its attack conjecture and corresponding response, thereby demonstrating in-context adaptation. Our agentic approach is free of modeling and can run on commodity hardware. When evaluated on incident logs reported in the literature, our agent achieves recovery up to 23% faster than those of frontier LLMs.

*   **核心痛点**：
    1.  **强化学习（RL）局限性**：现有的自动化响应多采用 RL，但其依赖手工建模的模拟器，且必须将丰富的系统日志语义压缩为简单的数值，丢失了关键信息。
    2.  **通用 LLM 局限性**：直接使用通用大模型进行响应规划时，容易出现“幻觉”（生成不可行的动作）和“上下文丢失”（在长程规划中忘记先前状态）。
*   **研究目标**：提出一种基于 14B 轻量级 LLM 的端到端智能体，利用其预训练的安全知识和上下文学习能力，在无需数学建模的情况下实现自主、自适应的网络事件响应。

### 2. 方法论
*   **核心思想**：将部分可观测马尔可夫决策过程（POMDP）的规划原理转化为 LLM 的智能体工作流，实现从原始日志到响应动作的直接映射。
*   **关键技术细节**：
    *   **四位一体架构**：
        1.  **感知（Perception）**：通过微调，使 LLM 能从非结构化日志中推断出 6 维布尔向量形式的“恢复状态”（包含遏制、评估、证据保存、驱逐、加固、恢复）。
        2.  **推理（Reasoning）**：构建内部“世界模型”，根据当前状态和假设的攻击战术（TTPs）预测未来的警报和状态演变。
        3.  **规划（Planning）**：采用类似蒙特卡洛树搜索（MCTS）的在线前瞻规划（Lookahead Rollout），模拟不同动作序列的后果并计算预期成本。
        4.  **行动（Action）**：将高层策略转化为具体的安全执行命令。
    *   **上下文自适应机制**：智能体会对比“预测警报”与“实际观察到的警报”，若不一致则触发校准程序，通过上下文学习更新对攻击模型的推测。
*   **算法流程**：采样 N 个候选动作 -> 对每个动作进行 M 次轨迹模拟 -> 选择累计恢复时间（成本）最小的动作执行 -> 接收新日志并校准模型。

*   **消融实验**：系统地移除“微调”、“规划”和“上下文自适应”模块，验证了各部分的必要性。

### 6. 主要结论与发现
*   **性能卓越**：该智能体生成的响应计划比 OpenAI o3 和 DeepSeek-R1 等前沿模型快 **23%**。
*   **状态推断精准**：在恢复状态的预测上，Exact-match 准确率达到 0.98，F1 分数接近 1.0。
*   **模块贡献**：微调和规划模块对缩短恢复时间贡献最大；上下文自适应在处理长程复杂攻击时具有重要价值。
*   **轻量化优势**：证明了经过领域特定微调的 14B 模型在特定任务（IR）上可以超越千亿参数规模的通用模型。

### 7. 优点
*   **语义保留**：直接处理原始文本日志，避免了传统方法中将语义信息转化为数值指标时的信息损失。
*   **端到端闭环**：实现了从感知到行动的完整闭环，且具备自我校准和纠错能力。
*   **硬件友好**：14B 的规模使其能够部署在普通商用硬件（如单块 A100）上，具有较强的实用性。

### 8. 不足与局限
*   **可扩展性瓶颈**：由于采用了 MCTS 搜索，计算复杂度随模拟次数和动作分支呈指数增长，20 分钟的响应时间在实时性要求极高的场景下仍显不足。

*   **长程挑战**：目前的测试数据多为短序列（约 5 步动作），在面对持续数周甚至数月的复杂持续性威胁（APT）时的表现尚待进一步验证。
