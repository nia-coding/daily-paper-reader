## 摘要
人类协作者通过过程可见性和工作空间感知进行动态协调，然而 AI 智能体通常要么仅提供最终输出，要么仅展示只读的执行过程（如规划、推理），而无法解释用户在共享制品上的并发操作。基于混合主动交互原则，我们探讨了智能体是否能够实现协作上下文感知——即解释用户在共享制品上的并发操作并实时进行调整。研究 1（N=10 名专业设计师）表明，过程可见性使得对智能体行为的推理成为可能，但当智能体无法区分反馈与独立工作时，会暴露冲突。我们开发了 CLEO，它能够解释协作意图并实时调整。研究 2（N=10，为期两天，包含刺激回忆访谈）分析了 214 个轮次，识别出五种动作模式、六种触发因素和四种赋能因素，解释了设计师何时选择委派（70.1%）、指导（28.5%）或并发工作（31.8%）。我们提出了一个包含六个交互循环的决策模型、设计启示以及一个标注数据集。

## Abstract
Human collaborators coordinate dynamically through process visibility and workspace awareness, yet AI agents typically either provide only final outputs or expose read-only execution processes (e.g., planning, reasoning) without interpreting concurrent user actions on shared artifacts. Building on mixed-initiative interaction principles, we explore whether agents can achieve collaborative context awareness -- interpreting concurrent user actions on shared artifacts and adapting in real-time. Study 1 (N=10 professional designers) revealed that process visibility enabled reasoning about agent actions but exposed conflicts when agents could not distinguish feedback from independent work. We developed CLEO, which interprets collaborative intent and adapts in real-time. Study 2 (N=10, two-day with stimulated recall interviews) analyzed 214 turns, identifying five action patterns, six triggers, and four enabling factors explaining when designers choose delegation (70.1%), direction (28.5%), or concurrent work (31.8%). We present a decision model with six interaction loops, design implications, and an annotated dataset.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：在人类协作中（如使用 Figma 或 Google Slides），成员会实时观察彼此的操作、预测下一步并随时介入。然而，目前的 AI 代理（Agent）大多遵循“提示-输出”的顺序循环，用户在 AI 执行任务时通常只能被动等待，无法在中间过程进行干预或同步协作。
*   **核心问题**：AI 代理是否能具备“协作上下文感知”（Collaborative Context Awareness），即在执行任务的同时，理解并适应用户对共享工件（Artifacts）的并发操作？
*   **整体含义**：论文旨在打破人机交互中的“回合制”限制，探索一种允许人类与 AI 在共享空间内同时工作的动态协作范式。

### 2. 方法论
研究者开发了一个名为 **Cleo (Collaborative Linked Executive Operator)** 的系统，其核心思想是赋予 Agent 识别和响应用户实时操作的能力。
*   **关键技术组件**：
    1.  **过程透明性（Process Transparency）**：Agent 不直接生成最终结果，而是通过调用 Figma API（如创建矩形、调整颜色）逐步展示操作过程，并配以虚拟光标。
    2.  **用户变更检测（User Change Detection）**：在 Agent 执行的每个微小步骤之间，系统会对比画布状态，识别出哪些变更是由用户手动完成的。
    3.  **归因变更模块（Attribution Change Module）**：区分 Agent 自身的动作结果与用户的并发修改。
    4.  **计划更新模块（Plan Update Module）**：如果检测到用户的操作与 Agent 原有计划冲突或提出了新方向，Agent 会实时调整其后续的执行逻辑。
*   **算法流程**：基于 **ReAct (Reasoning + Acting)** 框架。Agent 接收指令后生成计划，进入“推理-行动-反馈”循环。在每个循环中，Cleo 会整合用户的并发操作反馈，动态修正计划。

### 6. 主要结论与发现
*   **五种动作模式**：识别出用户在 Agent 执行时的行为：完全委派（70.1%）、观察监控（68.7%）、并发协作（31.8%）、指令引导（28.5%）和强制终止（8.9%）。
*   **六种触发因素**：用户介入的动机包括：灵感闪现、需要早期结果、准备进行细节微调、任务理解偏差、执行质量下降、产生新任务。
*   **决策模型**：提出了一个包含 **6 个交互循环** 的模型，解释了用户如何在“委派-观察-干预”之间流转。
*   **并发协作的价值**：并发交互不仅提高了效率，还减少了用户的“代理权丧失感”（Loss of Agency），使用户觉得 AI 是真正的合作伙伴。

### 7. 优点
*   **范式创新**：从“回合制”转向“并发制”，更符合真实世界的创意协作流程。
*   **技术落地**：Cleo 系统能够精准区分 Agent 动作与用户动作，解决了并发操作中的冲突问题。
*   **理论贡献**：详细的动作分类和决策模型为未来协作 AI 的 UX 设计提供了理论框架。

### 8. 不足与局限
*   **样本限制**：参与者仅限于 UI/UX 设计师，结论在编程、写作等其他领域的普适性有待验证。
*   **认知负荷**：观察 Agent 逐步执行并寻找介入时机可能会增加用户的监控负担。
*   **执行速度平衡**：Agent 执行太快用户来不及介入，太慢则效率低下，如何自动调节执行节奏（Pacing）仍是挑战。
*   **应用限制**：目前高度依赖 Figma 等具有成熟 API 的工具，在缺乏细粒度操作记录的软件中难以实现。
