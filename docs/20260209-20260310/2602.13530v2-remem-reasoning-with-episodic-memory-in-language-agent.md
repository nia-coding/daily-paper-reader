## 摘要
人类擅长记忆时空背景下的具体经历，并能跨事件进行推理，即具备情境记忆（episodic memory）的能力。相比之下，语言智能体的记忆主要仍是语义性的，目前的智能体尚无法有效地回忆和推理交互历史。我们针对这一差距识别并形式化了情境回忆与推理的核心挑战，并观察到现有工作往往忽视了情境性、缺乏显式的事件建模，或者过度强调简单检索而非复杂推理。我们提出了 REMem，一个用于构建情境记忆并进行推理的两阶段框架：1) 离线索引，REMem 将经历转换为混合记忆图谱，灵活地链接具有时间感知能力的要点（gists）和事实；2) 在线推理，REMem 采用一个配备精心设计工具的智能体检索器，在记忆图谱上进行迭代检索。在四个情境记忆基准测试上的全面评估表明，REMem 的性能显著优于 Mem0 和 HippoRAG 2 等最先进的记忆系统，在情境回忆和推理任务上分别实现了 3.4% 和 13.4% 的绝对提升。此外，REMem 在处理无法回答的问题时也表现出更稳健的拒绝行为。

## Abstract
Humans excel at remembering concrete experiences along spatiotemporal contexts and performing reasoning across those events, i.e., the capacity for episodic memory. In contrast, memory in language agents remains mainly semantic, and current agents are not yet capable of effectively recollecting and reasoning over interaction histories. We identify and formalize the core challenges of episodic recollection and reasoning from this gap, and observe that existing work often overlooks episodicity, lacks explicit event modeling, or overemphasizes simple retrieval rather than complex reasoning. We present REMem, a two-phase framework for constructing and reasoning with episodic memory: 1) Offline indexing, where REMem converts experiences into a hybrid memory graph that flexibly links time-aware gists and facts. 2) Online inference, where REMem employs an agentic retriever with carefully curated tools for iterative retrieval over the memory graph. Comprehensive evaluation across four episodic memory benchmarks shows that REMem substantially outperforms state-of-the-art memory systems such as Mem0 and HippoRAG 2, showing 3.4% and 13.4% absolute improvements on episodic recollection and reasoning tasks, respectively. Moreover, REMem also demonstrates more robust refusal behavior for unanswerable questions.

### 1. 核心问题与整体含义（研究动机和背景）
人类具备**情境记忆（Episodic Memory）**能力，能够沿着时空轴精确地重新经历过去的事件（即“心理时间旅行”），并理解事件间的顺序、持续时间和因果关系。
相比之下，当前的语言智能体（Language Agents）主要依赖**语义记忆（Semantic Memory）**。现有的 RAG 系统或结构化记忆系统（如 HippoRAG、Mem0）存在以下局限：
*   **缺乏显式事件建模**：往往将记忆视为扁平的向量空间或静态的事实三元组，忽略了时间、地点、参与者和情感等情境维度。
*   **推理能力不足**：大多依赖简单的相似度检索，难以处理涉及跨事件排序、计数或复杂逻辑的时间推理任务。
*   **时空脱节**：无法有效地将具体经历与时间轴绑定。

### 2. 方法论：REMem 框架
REMem 提出了一个两阶段框架，旨在赋予智能体类人的情境记忆能力：

#### A. 索引阶段（Indexing）—— 构建混合记忆图谱
1.  **要点提取（Gist Extraction）**：将原始交互（如对话）转化为简洁的、具有时间感知能力的自然语言“要点”（Gists）。每个要点包含绝对化的时间戳和关键情境要素（参与者、动作、地点等）。
2.  **事实提取（Fact Extraction）**：提取结构化的三元组（主-谓-宾），并附加 Wikidata 风格的时间限定符（如“发生时间”、“开始/结束时间”）。
3.  **混合图谱构建**：
    *   **节点**：包含“要点节点”（上下文层）和“短语节点”（概念层）。
    *   **边**：包括关系边（三元组逻辑）、上下文边（连接要点与其对应的事实）以及同义词边（基于 Embedding 相似度连接语义相近的要点）。

#### B. 代理式推理阶段（Agentic Inference）
采用 ReAct 风格的智能体，通过迭代调用工具来探索记忆图谱：
*   **检索工具**：语义检索（Embedding）和词法检索（BM25），用于获取初始种子节点。
*   **图探索工具**：`find_gist_contexts`（获取事件叙述）和 `find_entity_contexts`（根据实体和时间过滤条件查询事实）。
*   **流程控制**：智能体自主决定何时收集够了证据并输出最终答案。

### 6. 主要结论与发现
*   **性能领先**：REMem 在情境回忆上比 SOTA 提升 **3.4%**，在情境推理上提升 **13.4%**。
*   **推理能力突破**：在 Test of Time 挑战中，REMem 是唯一一个 Exact Match (EM) 分数**超过 90%** 的方法。
*   **拒绝行为更优**：相比 Graphiti 和 Mem0，REMem 在面对无法回答的问题时，能更准确地选择“拒绝回答”，具有更高的精确率和 F1 值。
*   **混合结构的价值**：要点（Gists）提供了主要的情境背景，而事实（Facts）在跨会话的逻辑连接中起到了关键的锚点作用。

### 7. 优点
*   **显式时间建模**：通过将时间戳和时间限定符集成到图结构中，解决了传统 RAG 难以处理时间维度的痛点。
*   **混合记忆表示**：结合了非结构化的自然语言要点和结构化的三元组，兼顾了叙述的完整性和逻辑的严密性。
*   **代理式迭代**：允许智能体像人类一样进行多步“回溯”和“联想”，而非一次性匹配。
*   **高效率**：相比于全上下文输入（Full-Context），REMem 在推理时消耗的 Token 显著减少，且表现更优。

### 8. 不足与局限
*   **索引成本**：构建记忆图谱需要多次调用 LLM 进行提取，对于超大规模实时流数据，索引的经济和时间成本可能较高。
*   **逻辑词敏感度**：错误分析显示，模型在处理“之前/之后”等细微的时间偏移方向时仍有约 18% 的错误率。
*   **静态索引限制**：目前主要针对离线批处理索引，对于实时流式（Streaming）记忆更新的工程实现仍有待进一步探索。
*   **依赖底层 LLM**：代理式推理的效果高度依赖于底层模型（如 GPT-4）的指令遵循和工具调用能力。
