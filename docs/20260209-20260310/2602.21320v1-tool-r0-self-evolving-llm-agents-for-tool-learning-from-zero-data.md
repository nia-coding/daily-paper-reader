## 摘要
大语言模型（LLMs）正成为能够使用工具解决复杂任务的自主智能体的基础。强化学习（RL）已成为注入此类智能体能力的常用方法，但通常处于严格受控的训练设置下。它往往依赖于精心构建的任务-解决方案对和大量的人工监督，这为迈向超智能系统的开放式自进化制造了根本性障碍。在本文中，我们提出了 Tool-R0 框架，在零数据假设下，通过自我博弈强化学习（self-play RL）从零开始训练通用工具调用智能体。Tool-R0 从同一个基础 LLM 初始化，通过互补奖励共同进化生成器（Generator）和求解器（Solver）：前者在后者的能力边界提出针对性的挑战任务，而后者则学习通过调用现实世界工具来解决这些任务。这创造了一个无需预先存在任务或数据集的自进化循环。在不同工具使用基准测试上的评估表明，Tool-R0 相比基础模型实现了 92.5% 的相对提升，并在相同设置下超过了全监督工具调用基线。我们的工作通过分析协同进化、课程动态和缩放行为，进一步为自我博弈 LLM 智能体提供了实证见解。

## Abstract
Large language models (LLMs) are becoming the foundation for autonomous agents that can use tools to solve complex tasks. Reinforcement learning (RL) has emerged as a common approach for injecting such agentic capabilities, but typically under tightly controlled training setups. It often depends on carefully constructed task-solution pairs and substantial human supervision, which creates a fundamental obstacle to open-ended self-evolution toward superintelligent systems. In this paper, we propose Tool-R0 framework for training general purpose tool-calling agents from scratch with self-play RL, under a zero-data assumption. Initialized from the same base LLM, Tool-R0 co-evolves a Generator and a Solver with complementary rewards: one proposes targeted challenging tasks at the other's competence frontier and the other learns to solve them with real-world tool calls. This creates a self-evolving cycle that requires no pre-existing tasks or datasets. Evaluation on different tool-use benchmarks show that Tool-R0 yields 92.5 relative improvement over the base model and surpasses fully supervised tool-calling baselines under the same setting. Our work further provides empirical insights into self-play LLM agents by analyzing co-evolution, curriculum dynamics, and scaling behavior.

### 2. 方法论：核心思想与技术细节
Tool-R0 框架通过**生成器（Generator）**和**求解器（Solver）**的协同进化实现自进化循环：
*   **核心思想**：初始化两个角色（均来自同一基础模型），生成器负责设计具有挑战性的工具使用任务，求解器负责解决这些任务。通过互补的奖励机制，生成器不断寻找求解器的“能力边界”，从而提供动态的课程学习。
*   **关键技术细节**：
    *   **接地任务规范（Grounded Task Specification）**：为了防止生成器陷入模式坍塌（Mode Collapse），通过采样领域、上下文、工具数量等配置来约束生成过程。
    *   **生成器奖励设计**：
        *   **格式奖励 ($r_{fmt}$)**：确保输出符合 JSON 等可解析格式。
        *   **有效性奖励 ($r_{valid}$)**：检查生成的工具调用是否在菜单中、参数是否完整、参数值是否在问题中出现。
        *   **课程奖励 ($r_{curr}$)**：这是核心。利用**难度引导奖励 ($r_{diff}$)**，通过蒙特卡洛采样评估求解器的成功率。它采用带通滤波器（Band-pass signal），优先奖励那些求解器胜率在 25%-75% 之间的任务（即“不太难也不太易”的任务）。
    *   **求解器训练**：使用 GRPO 算法，基于生成器产出的任务进行强化学习，奖励基于工具调用的准确性（名称、键、值的匹配度）。
    *   **自进化循环**：训练生成器 -> 合成并筛选任务数据集 -> 训练求解器 -> 进入下一轮迭代。

### 6. 主要结论与发现
*   **零数据可行性**：Tool-R0 在零人工数据下，使 1.5B 模型的工具调用能力提升了 92.5%，甚至超过了使用数十万条人工数据的监督模型。
*   **角色分离必要性**：实验证明，生成器和求解器必须使用独立的参数，否则会因为奖励目标冲突导致性能大幅下降（下降约 36%）。
*   **自生成课程优于人工**：自博弈产生的任务分布比静态的人工数据集更广泛、更平衡，能更精准地针对模型自身的弱点。
*   **中段训练（Mid-training）潜力**：Tool-R0 可以作为一种有效的预训练增强手段，提升后续 SFT 的效果。

### 7. 优点（亮点）
*   **完全脱离人工数据**：实现了从 0 到 1 的工具能力构建，具有极强的扩展性。
*   **动态难度对齐**：通过带通滤波器奖励函数，巧妙地实现了自动化的课程学习，确保模型始终在“舒适区”边缘学习。
*   **泛化能力强**：不仅在合成任务上表现好，在未见过的真实世界 API 基准上也表现出极强的鲁棒性。

### 8. 不足与局限
*   **小模型饱和**：在 0.5B 等极小规模模型上，自博弈在 3 轮后迅速达到饱和，提升空间受限。
*   **计算开销**：难度评估需要对求解器进行多次采样（K=8），这增加了训练时的推理成本。
*   **知识边界限制**：由于生成器和求解器源自同一模型，其生成的任务复杂度受限于模型自身的潜在知识边界，可能难以凭空产生模型完全未接触过的领域知识。
*   **奖励黑客风险**：虽然有语义一致性检查，但在极少数情况下，模型仍可能通过生成看似正确但逻辑有误的任务来“刷分”。
