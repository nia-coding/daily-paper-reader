## 摘要
在这项工作中，我们识别并解决了 LLM 推理服务中智能体记忆管理的核心挑战。在这些场景下，大规模存储、频繁更新以及多个智能体共存共同引入了复杂且高成本的近似最近邻（ANN）搜索问题。我们提出了 Pancake，这是一个多层级智能体记忆系统，它统一了三项关键技术：(i) 面向单个智能体的多级索引缓存，(ii) 跨多个智能体的协调索引管理，以及 (iii) GPU-CPU 协同加速。Pancake 提供了易于使用的接口，可集成到 Mem-GPT 等基于记忆的智能体中，并兼容 LangChain 和 LlamaIndex 等智能体框架。在真实智能体工作负载上的实验表明，Pancake 的性能显著优于现有框架，实现了超过 4.29 倍的端到端吞吐量提升。

## Abstract
In this work, we identify and address the core challenges of agentic memory management in LLM serving, where large-scale storage, frequent updates, and multiple coexisting agents jointly introduce complex and high-cost approximate nearest neighbor (ANN) searching problems. We present Pancake, a multi-tier agentic memory system that unifies three key techniques: (i) multi-level index caching for single agents, (ii) coordinated index management across multiple agents, and (iii) collaborative GPU-CPU acceleration. Pancake exposes easy-to-use interface that can be integrated into memory-based agents like Mem-GPT, and is compatible with agentic frameworks such as LangChain and LlamaIndex. Experiments on realistic agent workloads show that Pancake substantially outperforms existing frameworks, achieving more than 4.29x end-to-end throughput improvement.

### 1. 核心问题与整体含义（研究动机和背景）
随着大模型智能体（Agent）向长程对话、复杂任务规划和多智能体协作演进，**智能体记忆（Agent Memory）**成为维持一致性和推理质量的关键。与传统的检索增强生成（RAG）不同，智能体记忆具有**高度动态性**：需要频繁地交替进行搜索（Search）和插入/更新（Update）。
*   **现有挑战**：现有的向量数据库（如 Faiss）或智能体框架（如 MemGPT）在处理大规模动态记忆时，搜索开销随记忆规模剧增（有时占总执行时间的 82% 以上）。
*   **技术瓶颈**：
    1.  **单智能体更新低效**：频繁的小批量插入会导致向量散落在不相关的聚类中（Scattered Cluster Problem），降低检索召回率和效率。
    2.  **多智能体索引冗余**：为每个智能体维护独立索引会导致跨智能体搜索时粗粒度搜索（Coarse Search）开销倍增。
    3.  **硬件利用不均**：GPU 显存受限且难以处理频繁的动态索引更新，导致 CPU-GPU 协同效率低下。

### 2. 方法论：核心思想与关键技术
Pancake 提出了一个多层级的存储管理系统，核心思想是**利用智能体访问模式的局部性（Locality）来优化动态索引**。

*   **多级索引缓存（Multi-level Index Cache）**：
    *   设计 L0（最近访问）、L1（邻域缓存）、L2（稳定全局索引）三层结构。
    *   **FSM 模式建模**：将智能体的记忆访问抽象为有限状态机（FSM），捕捉“步骤间”的语义转换规律，实现精准的预取（Prefetching）和搜索重排序。
*   **混合图索引（Hybrid Graph Indexing）**：
    *   针对多智能体场景，将多个智能体的私有索引与全局静态知识索引通过“门户节点”连接，形成一个**混合图**。
    *   **智能体画像（Agent Profile）**：在共享聚类中记录特定智能体的访问偏好，减少无关向量的扫描。
*   **GPU-CPU 协同管理**：
    *   **热点缓存**：将高频访问的聚类驻留在 GPU。
    *   **插入缓冲区（Insertion Buffer）**：新插入的向量先进入 CPU 缓冲区，搜索时并行计算 CPU 缓冲区和 GPU 缓存结果，避免频繁的显存同步。
    *   **异步一致性**：利用异步传输和 GPU 端的 K-means 聚类分裂，降低在线更新的停顿感。
