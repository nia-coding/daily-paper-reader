## 摘要
长上下文大语言模型（LLM）和检索增强生成（RAG）系统被动地处理信息，将状态追踪、矛盾解决和证据聚合推迟到查询阶段，这在包含频繁更新的超长流式数据下变得脆弱。我们提出了统一记忆智能体（Unified Memory Agent, UMA），这是一个端到端强化学习框架，在单一策略中统一了记忆操作和问答。UMA 维持一种双重记忆表示：用于全局上下文的紧凑核心摘要，以及支持对键值条目进行显式 CRUD（创建、更新、删除、重组）操作的结构化记忆库（Memory Bank），从而在流式处理过程中实现主动整合。为了评估长程记忆行为，我们引入了 Ledger-QA，这是一个用于连续状态追踪的诊断基准，其中的答案是源自累积更新的潜在值，而非局部的片段检索。在涵盖 Ledger-QA、测试时学习（Test-Time Learning）和准确检索的 13 个数据集上，UMA 在动态推理和学习任务上显著优于长上下文和 RAG 基线，同时在标准检索基准上保持了竞争力，强调了学习型端到端记忆管理的重要性。

## Abstract
Long-context LLMs and Retrieval-Augmented Generation (RAG) systems process information passively, deferring state tracking, contradiction resolution, and evidence aggregation to query time, which becomes brittle under ultra long streams with frequent updates. We propose the Unified Memory Agent (UMA), an end-to-end reinforcement learning framework that unifies memory operations and question answering within a single policy. UMA maintains a dual memory representation: a compact core summary for global context and a structured Memory Bank that supports explicit CRUD (create, update, delete, reorganize) over key value entries, enabling proactive consolidation during streaming. To evaluate long-horizon memory behavior, we introduce Ledger-QA, a diagnostic benchmark for continuous state tracking where answers are latent values derived from accumulated updates rather than lo cal span retrieval. Across 13 datasets spanning Ledger-QA, Test-Time Learning, and Accurate Retrieval, UMA substantially outperforms long-context and RAG baselines on dynamic reasoning and learning tasks while remaining competitive on standard retrieval benchmarks, underscoring the importance of learned, end-to-end memory management.

### 论文总结：学习记忆——用于长上下文推理的记忆智能体端到端训练

#### 1. 核心问题与整体含义（研究动机和背景）
当前长上下文大语言模型（LLM）和检索增强生成（RAG）系统在处理超长信息流时存在**“被动处理”**的缺陷。它们通常将状态追踪、矛盾解决和证据聚合推迟到查询阶段（Query Time），这导致了“上下文腐烂”（Context Rot）现象：随着输入增加，模型难以定位关键细节。此外，RAG在每次查询时重新处理原始片段，计算冗余且难以处理需要跨时间步累积信息的动态任务。
本研究旨在开发一种**主动记忆智能体（UMA）**，使其能够在信息到达时主动进行整合、更新和清理，从而实现高效的长程动态推理。

#### 2. 方法论：核心思想与技术细节
论文提出了 **统一记忆智能体（Unified Memory Agent, UMA）** 框架，其核心思想是将长上下文推理建模为马尔可夫决策过程（MDP），并通过端到端强化学习优化记忆管理。

*   **双重记忆表示**：
    *   **核心摘要（Core Summary）**：提供全局上下文的高级压缩描述。
    *   **结构化记忆库（Memory Bank）**：支持显式的 **CRUD**（创建、读取、更新、删除）操作，以键值对（Key-Value）形式存储精确信息。
*   **两阶段运行流程**：
    *   **阶段 I（记忆维护）**：逐块处理输入流，智能体自主决定是否添加、更新或删除记忆库中的条目，最后通过 `UpdateCore` 动作压缩进度。
    *   **阶段 II（混合检索问答）**：在查询阶段，智能体结合结构化记忆库检索和原始文本的混合检索（BM25/Embedding）来生成答案。
*   **关键算法：任务分层 GRPO（Task-Stratified GRPO）**：
    *   针对记忆操作（长期效用）和问答操作（短期结果）奖励异构的问题，改进了 GRPO 算法。
    *   **分层优势估计**：将记忆步骤的优势值基于未来所有相关问题的平均奖励进行归一化，而将 QA 步骤的优势值仅在同问题的不同轨迹间归一化，确保了信用分配（Credit Assignment）的公平性。

#### 3. 实验设计
研究在 13 个数据集上进行了广泛评估，涵盖三大类任务：
*   **动态状态追踪（新基准：Ledger-QA）**：论文自研的合成基准，模拟长达一年的个人财务管理。要求智能体在嘈杂的对话流中提取交易并持续更新余额。答案是累积计算的结果，而非简单的文本检索。
*   **测试时学习（Test-Time Learning, TTL）**：如 Banking77、Clinic 等，要求智能体在上下文中学习分类规则并应用于新样本。
*   **准确检索（Accurate Retrieval, AR）**：如 HotpotQA、SQuAD、MSC 等标准长文本/多轮对话检索任务。
*   **对比方法**：包括全文本拼接（Concat）、标准 RAG、循环摘要模型（MemAgent）、以及先进的记忆智能体（Mem1, MemAlpha）。

#### 4. 资源与算力
*   **硬件环境**：使用了由 **32 张 NVIDIA H200 GPU** 组成的集群（4 个节点，每节点 8 卡）。
*   **模型基础**：策略网络初始化自 `Qwen3-4B-Instruct`。
*   **训练时长**：
    *   通用智能体（Generalist Agent）：在混合数据集上训练约 **2 天**。
    *   专用智能体（Specialist Agent）：在 Ledger-QA 上微调约 **0.5 天**。
*   **推理配置**：使用 vLLM 框架，设置最大序列长度为 8,192 标记。

#### 5. 实验数量与充分性
实验设计非常充分且具有说服力：
*   **多维度评估**：涵盖了从 2 个 Session 到 50 个 Session 的不同长度跨度，验证了模型在超长程（40k+ tokens）下的稳定性。
*   **消融实验**：对比了“无强化学习”、“无记忆维护阶段”、“全局奖励分组”以及“两阶段非端到端训练”等变体，深入探讨了各组件的贡献。
*   **客观性**：采用了 LLM-as-a-Judge（使用 Qwen3-30B 作为裁判）并结合了 Exact Match、F1、ROUGE-L 等硬性指标，确保了评估的客观性。

#### 6. 主要结论与发现
*   **主动记忆的优越性**：UMA 在 Ledger-QA 这种需要持续状态更新的任务上大幅领先。在 50 个 Session 的极端情况下，UMA 保持了 50% 以上的准确率，而 RAG 和其他基线跌至 20% 以下。
*   **端到端训练的必要性**：消融实验证明，端到端联合优化记忆和 QA 显著优于将两者分开训练。这使得记忆操作能够“预见”未来的查询需求。
*   **泛化能力**：尽管在特定任务上训练，UMA 在未见过的 TTL 任务（如 Banking77）上也表现出极强的上下文学习能力。

#### 7. 优点（亮点）
*   **范式转变**：将长上下文处理从“被动检索”转变为“主动管理”，更符合人类处理长期信息的逻辑。
*   **算法创新**：提出的 Task-Stratified GRPO 有效解决了记忆智能体训练中异构任务奖励难以对齐的痛点。
*   **高质量基准**：Ledger-QA 填补了现有长文本基准中缺乏“动态状态追踪”评估的空白。

#### 8. 不足与局限
*   **硬性窗口限制**：目前实现中仍存在 16k 标记的硬性上下文窗口截断，虽然记忆库缓解了信息丢失，但在极端无限长场景下的表现仍有待探索。
*   **合成数据偏差**：Ledger-QA 虽然严谨，但属于程序化生成的合成数据，真实世界的财务记录或状态追踪可能包含更多非结构化噪声和歧义。
*   **模态限制**：目前仅支持文本信息，尚未扩展到多模态记忆（如图像、音频流）的管理。
