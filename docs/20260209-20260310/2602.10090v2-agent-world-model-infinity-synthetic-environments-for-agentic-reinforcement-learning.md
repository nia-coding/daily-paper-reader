## 摘要
大语言模型（LLM）的最新进展使自主智能体能够执行需要与工具和环境进行多轮交互的复杂任务。然而，由于缺乏多样化且可靠的环境，扩展此类智能体训练受到了限制。在本文中，我们提出了智能体世界模型（Agent World Model, AWM），这是一个全合成环境生成流水线。利用该流水线，我们扩展到了覆盖日常场景的 1,000 个环境，在这些环境中，智能体可以与丰富的工具集（平均每个环境 35 个工具）进行交互并获得高质量的观测。值得注意的是，这些环境是由代码驱动并由数据库支持的，与由 LLM 模拟的环境相比，提供了更可靠且一致的状态转换。此外，与从现实环境中收集轨迹相比，它们能够实现更高效的智能体交互。为了证明该资源的有效性，我们针对多轮工具使用智能体进行了大规模强化学习。得益于完全可执行的环境和可访问的数据库状态，我们还可以设计可靠的奖励函数。在三个基准测试上的实验表明，仅在合成环境中进行训练（而非特定于基准测试的环境），可以产生强大的分布外泛化能力。代码可在 https://github.com/Snowflake-Labs/agent-world-model 获取。

## Abstract
Recent advances in large language model (LLM) have empowered autonomous agents to perform complex tasks that require multi-turn interactions with tools and environments. However, scaling such agent training is limited by the lack of diverse and reliable environments. In this paper, we propose Agent World Model (AWM), a fully synthetic environment generation pipeline. Using this pipeline, we scale to 1,000 environments covering everyday scenarios, in which agents can interact with rich toolsets (35 tools per environment on average) and obtain high-quality observations. Notably, these environments are code-driven and backed by databases, providing more reliable and consistent state transitions than environments simulated by LLMs. Moreover, they enable more efficient agent interaction compared with collecting trajectories from realistic environments. To demonstrate the effectiveness of this resource, we perform large-scale reinforcement learning for multi-turn tool-use agents. Thanks to the fully executable environments and accessible database states, we can also design reliable reward functions. Experiments on three benchmarks show that training exclusively in synthetic environments, rather than benchmark-specific ones, yields strong out-of-distribution generalization. The code is available at https://github.com/Snowflake-Labs/agent-world-model.

#### 1. 核心问题与整体含义（研究动机和背景）
当前自主智能体（Agents）的训练面临严重的环境瓶颈：
*   **环境稀缺且难以扩展**：现实世界的 API 调用昂贵且不稳定，而人工创建的环境（如 $\tau^2$-bench）规模极小（仅 3-5 个场景），不足以支撑大规模强化学习（RL）。
*   **模拟环境不可靠**：现有的 LLM 模拟环境（由模型生成状态转移）存在严重的“幻觉”问题，且推理成本高、效率低。
*   **泛化性不足**：智能体往往在特定基准测试上过拟合，缺乏在未见过的（OOD）场景中执行多轮工具调用的能力。

**AWM 的核心意义**在于提出了一个全自动、可扩展的流水线，生成了 1,000 个由代码驱动、数据库支持的真实感合成环境，为智能体的大规模强化学习提供了“无限”的训练场。

#### 2. 方法论：核心思想与关键技术
AWM 将环境合成分解为五个系统化步骤，模拟软件开发的实际流程：
1.  **场景生成（Scenario Synthesis）**：从 100 个种子域名出发，利用 LLM 生成 1,000 个涵盖电商、金融、社交等领域的场景描述，并通过嵌入向量去重确保多样性。
2.  **任务合成（Task Synthesis）**：为每个场景生成约 10 个具体的、可由 API 解决的用户任务（共 10,000 个任务）。
3.  **数据库设计与数据合成**：利用 SQLite 作为后端，生成符合任务需求的数据库模式（Schema）并填充初始样本数据，确保状态转换的**一致性**。
4.  **接口合成（Interface Synthesis）**：生成基于 Python 的工具集，并通过 **Model Context Protocol (MCP)** 暴露接口，定义智能体的动作空间和观测空间。
5.  **验证与强化学习**：
    *   **代码增强的 LLM-as-a-Judge**：结合数据库状态差异（State Diff）和 LLM 推理来提供鲁棒的奖励信号。
    *   **GRPO 算法**：采用群体相对策略优化（Group Relative Policy Optimization）进行在线 RL。
    *   **历史感知训练（History-Aware Training）**：在训练中应用与推理时一致的历史截断（Sliding Window），解决训练与部署时的分布偏移。

#### 6. 主要结论与发现
*   **合成环境的有效性**：仅在合成环境上训练，智能体在 BFCLv3 和 MCP-Universe 等真实基准上的表现显著提升（如 8B 模型在 BFCLv3 上从 53.83 提升至 65.94）。

*   **奖励鲁棒性**：结合数据库状态的 LLM 裁判比单一的验证方式更有效，能容忍合成环境中的微小 Bug。

#### 7. 优点
*   **高可靠性**：通过 SQL 数据库和 Python 代码确保了状态转换的逻辑严密性，克服了 LLM 模拟器的幻觉。
*   **大规模与标准化**：提供了目前最大的开源工具使用环境集（1,000 个），并采用了 MCP 这一新兴的行业标准接口。
*   **全自动流水线**：从场景名到验证代码全自动生成，极大地降低了环境构建成本（平均每个环境合成成本约 57 美元）。

#### 8. 不足与局限
*   **环境 Bug 风险**：尽管有自我修复机制，但约 74%-83% 的合成环境仍存在细微 Bug（如未处理边缘情况），可能影响训练信号。
*   **场景覆盖限制**：目前主要集中在 CRUD（增删改查）类应用，尚未覆盖需要复杂 GUI 交互或长程信息检索（RAG）的场景。
*   **静态生成**：环境生成后是静态的，尚未实现智能体与环境生成器之间的“自我进化”闭环。
*   **算力开销**：大规模在线 RL 配合 LLM 裁判仍需较高的计算资源和 API 调用成本。
